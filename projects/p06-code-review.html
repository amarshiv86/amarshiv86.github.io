AI Code Review Bot

Automated pull request reviewer using Mistral 7B via HuggingFace Inference API. Triggered by GitHub Actions on every PR — fetches the diff, generates structured inline comments with severity ratings, and posts them directly to the PR via GitHub API.

Architecture
How it works
  • 1
    PR opened or updated — GitHub triggers the configured webhook, sending a signed payload to the bot's /webhook endpoint.
  • 2
    HMAC-SHA256 verification — the bot verifies the X-Hub-Signature-256 header using the shared WEBHOOK_SECRET before processing. Invalid signatures return 401.
  • 3
    Fetch diff — GitHubClient calls the GitHub API to get the full PR diff. Diffs over 12,000 chars are truncated with a warning log, preserving the most important changes.
  • 4
    Mistral 7B review — diff is sent to Mistral-7B-Instruct via HF Inference API with a structured JSON prompt. Temperature 0.1 for consistent output. Returns severity + per-line comments.
  • 5
    JSON parsing + validation — response is stripped of markdown fences, JSON extracted, validated for required fields with safe defaults. Malformed responses don't crash the server.
  • 6
    Post inline comments — GitHubClient posts each comment as a GitHub PR review inline comment at the specific file + line. Overall severity maps to REQUEST_CHANGES (critical/major) or COMMENT (minor/info).
Severity System

Every comment gets a severity rating. The overall PR severity maps to GitHub's review event — critical/major trigger REQUEST_CHANGES (blocks merge), minor/info post as COMMENT (informational):

🔴 Critical
Security vulnerability, data loss risk, crashes — REQUEST_CHANGES
🟠 Major
Logic bug, significant perf issue, broken functionality — REQUEST_CHANGES
🔵 Minor
Code smell, unclear naming, missing error handling — COMMENT
⚪ Info
Style suggestion, minor improvement — COMMENT

Sample review JSON returned by Mistral:

{
  "overall_severity": "major",
  "summary": "SQL injection vulnerability in user input handling.
             Missing input validation on line 42.",
  "comments": [
    {
      "path": "api/routes/users.py",

SRE Additions
HMAC-SHA256 webhook auth — verifies every GitHub request before processing
/health endpoint — uptime minutes + reviews_run counter, ping with UptimeRobot
Structured logging — per-review latency, comment count, severity all logged
Diff truncation — large diffs truncated to 12k chars with warning, never crashes
JSON validation — malformed model responses get safe defaults, not 500 errors
Review event mapping — severity → GitHub REQUEST_CHANGES vs COMMENT
Low temperature — 0.1 temperature for consistent structured JSON output
Repetition penalty — 1.1 penalty prevents model looping in long responses
Key Design Decisions
Why structured JSON output instead of free-form text?
Free-form review text can't be programmatically parsed into GitHub inline comments — you can't know which file or line number a comment refers to. Structured JSON with path, line, severity, and body fields maps directly to the GitHub PR Review API. The system prompt explicitly forbids any text outside the JSON object.
Why temperature 0.1 for code review?
High temperature produces creative but inconsistent output — a security issue might get flagged in one run and missed in the next. Code review requires deterministic analysis. Temperature 0.1 keeps the model focused on the actual code rather than generating varied creative commentary.
Why HMAC-SHA256 webhook verification?
Without verification, anyone who knows your webhook URL can trigger the bot with arbitrary payloads — potentially causing it to post reviews on repos it shouldn't access, or consuming your HF API quota. GitHub signs every webhook payload with the shared secret; verifying this signature ensures only GitHub can trigger reviews.
Why Mistral 7B instead of a smaller model?
Code review requires understanding context across multiple files, identifying logic bugs, and generating actionable fix suggestions. Sub-3B models struggle with this — they tend to produce generic comments ("add error handling") without referencing specific code. Mistral 7B is the smallest model that reliably produces useful, code-specific feedback.

# Run bot locally
git clone https://github.com/amarshiv86/p06-ai-code-review
cd p06-ai-code-review
pip install -r requirements.txt
cp .env.example .env  # fill HF_TOKEN, GITHUB_TOKEN, WEBHOOK_SECRET

uvicorn src.main:app --reload --port 8000

# Expose to GitHub webhooks via ngrok
ngrok http 8000
# Copy the https URL → GitHub repo Settings → Webhooks
# Set Content-Type: application/json
# Set Secret: same value as WEBHOOK_SECRET
# Select: Pull request events only

# Health check
curl http://localhost:8000/health
# → {"status":"ok","uptime_minutes":2.1,"reviews_run":0}

# Run tests
pytest tests/ -v
